MCP Connectors

Extend AURA with the Model Context Protocol — connect to external services, data sources, and custom tools through a unified interface.

Overview

The Model Context Protocol (MCP) is an open standard that lets AI agents discover and interact with external tools and data sources. Instead of hard-coding integrations, AURA uses MCP as a universal adapter layer — any service that speaks MCP can be plugged in without modifying the core agent.

MCP servers expose resources (readable data) and tools (callable actions). AURA discovers these automatically and decides when to use them based on your prompt.

Note: MCP is an open protocol developed by Anthropic. Any MCP-compatible server — whether written in Python, TypeScript, Go, or Rust — works with AURA out of the box.

Built-in Connectors

AURA supports the Model Context Protocol natively. An example GitHub connector configuration is included but disabled by default. You can add any MCP-compatible server by editing the config file.

GitHub (disabled by default)

Browse repositories, read files, create issues, review pull requests, and manage branches. AURA authenticates via a personal access token configured in Settings under MCP > GitHub.

Tip: The GitHub connector requires network permission. If you are in a restricted environment, enable it in Settings > Permissions before first use.

Custom MCP Servers

You can add any MCP-compatible server by providing its command and arguments. AURA launches the server as a subprocess and communicates over stdio.

Configure custom servers in mcp_servers.json inside the AURA config directory:

{
  // File: ~/.aura/mcp_servers.json
  "mcpServers": {
    "sqlite": {
      "command": "uvx",
      "args": ["mcp-server-sqlite", "--db-path", "~/.aura/data.db"]
    },
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/Users/Shared"
      ]
    },
    "custom-api": {
      "command": "python",
      "args": ["-m", "my_mcp_server"],
      "env": {
        "API_KEY": "${env:MY_API_KEY}"
      }
    }
  }
}

Connecting

  1. Open AURA and navigate to Settings > MCP.
  2. Toggle the GitHub connector on and complete authentication if prompted.
  3. For custom servers, click Add Server and enter the command and arguments, or edit mcp_servers.json directly.
  4. Restart the AURA server or click Refresh in the MCP panel to load new servers.
  5. Start a chat and ask AURA to use a connected tool — it will discover available tools automatically.

Verify a server is running by checking the MCP status indicator in the sidebar. Green means connected; red indicates the server failed to start.

Environment Variables

You can inject environment variables into custom MCP servers using the ${env:VAR_NAME} syntax. This keeps secrets out of your config file.

{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "${env:GITHUB_TOKEN}"
      }
    }
  }
}

Use Cases

  • Automated code review: Connect to GitHub, point AURA at an open PR, and ask it to review the diff and leave comments.
  • Meeting summaries: Pull messages from a messaging channel via MCP, summarize the discussion, and save the result to a document.
  • Data analysis: Query a local SQLite database through MCP and have AURA generate charts from the results.
  • DevOps triage: Check GitHub Actions status, read logs, and restart failed workflows — all from a single prompt.
  • Document management: Search a document store for files matching a description, download them, and process them with AURA build mode.
Tip: MCP servers run as child processes of AURA. If a server crashes, AURA restarts it automatically. Check the server's stderr output in the MCP panel for debugging.
