Terminal

Grant AURA an interactive shell session to run commands, inspect output, and automate server-side workflows — no GUI required.

Overview

The Terminal capability gives the model a persistent pseudo-terminal (PTY) session on the server. Unlike the one-shot script execution in Computer Control, a Terminal session preserves state across commands — current directory, environment variables, shell history, and running processes all carry over.

This is ideal for multi-step tasks like cloning a repository, installing dependencies, running tests, or any workflow that benefits from an interactive shell.

How It Works

When AURA creates a terminal session, it spawns a real PTY-backed shell process (bash, zsh, or PowerShell). The model can then:

  • Execute a command and read its combined stdout/stderr output
  • Write data directly to the shell's stdin (for interactive programs)
  • Read any buffered output since the last read
  • Resize the PTY dimensions for programs that check terminal size
  • Close the session when done
$ "Clone the repo and install dependencies."
$ "Run the test suite and show me the results."
$ "Start the dev server in the background."
$ "What processes are using port 3000?"

Session Lifecycle

Sessions are created on demand and persist until explicitly closed or the server shuts down. Each session holds its own shell state, so you can run multiple independent terminals. The server also closes all remaining sessions during graceful shutdown.

Note: Terminal sessions are server-side only. There is no visible terminal window on the desktop — the PTY runs in the background. If you need visible GUI automation, use Computer Control instead.

API Endpoints

The Terminal capability exposes REST endpoints under /v1/terminal/:

  • POST /v1/terminal/create — Create a new session (body: {"shell": "bash"})
  • POST /v1/terminal/exec?session_id={id} — Execute a command (body: {"command": "ls", "timeout": 30})
  • GET /v1/terminal/{id}/read?timeout=1.0 — Read buffered output
  • POST /v1/terminal/{id}/write?data=...{} — Write to stdin
  • POST /v1/terminal/{id}/resize — Resize PTY (body: {"rows": 24, "cols": 80})
  • POST /v1/terminal/{id}/close — Close a session
  • GET /v1/terminal/sessions — List all active sessions

Cross-Platform Support

TerminalAgent works on all three platforms:

  • macOS / Linux: Uses PTY via Python's pty module for full terminal emulation
  • Windows: Falls back to subprocess.Popen pipes (no PTY on Windows)

Prerequisites

No special permissions are needed. The terminal runs as the same user as the AURA server process, inheriting its environment and filesystem access.

Tip: Combine Terminal with Build Mode for a complete development workflow: clone, build, test, and iterate all within the same session.

Common Use Cases

# Git workflow
$ "Clone the repo, checkout a new branch, and push an empty commit."
$ "Show me the last 5 commits in this repo."
# Package management
$ "Install numpy, pandas, and matplotlib with pip."
$ "Update all npm packages to their latest versions."
# System monitoring
$ "How much disk space is free?"
$ "Find the process using the most memory."
# Development servers
$ "Start the Flask dev server on port 5000 in the background."
$ "Check if the server is responding with curl."
Privacy: Terminal commands run entirely on your machine. No output or session data is sent to external services.
