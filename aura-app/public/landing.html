FAQ

Common questions

What is AURA? ▼
AURA is a local-first AI agent that runs entirely on your machine. It provides multi-model chat, computer control, app automation, video editing, and more — all through a single interface. No data leaves your machine unless you explicitly choose to connect to a cloud model.
How do I get started? ▼
Clone the repository, run the start script, and open the app. AURA defaults to Ollama if available; otherwise it falls back to the built-in GPT-2 model. No API keys required. For more capable models, you can connect to OpenAI or Gemini.
Do I need extra AI subscriptions? ▼
No. AURA runs fully locally and defaults to Ollama (if available) with GPT-2 as fallback. If you want better performance, you can bring your own API keys for OpenAI or Gemini — but none are required.
What about data and privacy? ▼
AURA is built for privacy first. All processing happens locally. There is no telemetry, no analytics, and no data collection. Every capability that touches your system (screen recording, file access, automation) requires explicit permission that you can grant or revoke at any time.
Is AURA open source? ▼
Yes. AURA is fully open source. You can inspect, modify, and distribute the code. Contributions are welcome.
Which platforms are supported? ▼
AURA supports macOS, Windows, and Linux. The app runs on any platform with Python 3.9+ and a modern web browser. Computer control features use platform-native accessibility APIs (Accessibility API on macOS, UI Automation on Windows, X11/Wayland on Linux).
