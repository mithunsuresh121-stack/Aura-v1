Permissions & Security

CORTEX uses a granular permission model to protect your system. Every capability — from screen recording to network access — must be explicitly granted before it can be used.

Overview

CORTEX's permission model is built on the principle of least privilege. No capability ever receives more access than necessary, and every permission must be explicitly opted into by the user. Permissions are stored locally and enforced by the CORTEX backend at runtime. When a capability requires a permission that has not been granted, CORTEX prompts you with a detailed grant dialog explaining exactly what is being requested and why.

You can manage all permissions from the Settings panel inside the app, under the Permissions tab. Permissions persist across sessions until explicitly revoked.

The 9 Permissions

Permissions are organised into four groups. Each permission controls a specific subsystem of CORTEX.

Permission Group Purpose
screen_recording System Capture screen contents for computer-control and vision tasks
accessibility System Control keyboard, mouse, and UI elements via accessibility APIs
automation System Execute scripts, launch processes, and automate workflows
script_execution System Run arbitrary shell scripts and terminal commands
file_system Files Read and write files across permitted directories
video_editing Creative Control DaVinci Resolve, Premiere Pro, and CapCut timelines
photo_editing Creative Future capability — planned for photo editing automation
sound_editing Creative Future capability — planned for audio editing automation
network_access Network Send requests to cloud model providers and external services

Granting and Revoking

Permissions can be granted or revoked at any time from the app's Settings panel — Permissions tab. Each permission has its own toggle switch. Flip it on to grant, off to revoke. Changes take effect immediately; no restart required.

When a capability first requests a permission you haven't granted, CORTEX displays a grant dialog with a brief explanation of what the permission is used for. You can approve, deny, or choose to be reminded later.

You can manage permissions at three granularity levels:

  • Single — toggle individual permissions one at a time for fine-grained control.
  • Group — each group (System, Files, Creative, Network) has a group toggle that grants or revokes every permission in that category simultaneously.
  • All — click Revoke All in the Permissions tab to reset every permission to the "not granted" state, forcing all future capability requests to prompt again.

Privacy Guarantees

CORTEX is designed with privacy as a core principle:

  • All permissions are stored locally on your machine — never sent to any server.
  • Permission state is checked at the backend layer; the frontend cannot invoke a capability that has not been granted.
  • Screen Recording and Accessibility permissions are only active while a computer-control task is running. CORTEX does not passively monitor your screen or input.
  • File System access is scoped to the directories you explicitly allow — CORTEX never recursively walks your entire filesystem.
  • Network access is only used when you explicitly configure a model provider and send a request. CORTEX never phones home without your knowledge.
  • No telemetry, analytics, or usage data is collected. CORTEX is fully offline by design.
Note: On macOS, Screen Recording and Accessibility require system-level approval in addition to CORTEX's own permissions. Open System Settings → Privacy & Security and grant CORTEX access for both. Without this, the in-app permission toggle will have no effect.
Tip: Review your permissions regularly. The Permissions tab shows the current state of every permission at a glance. Revoke any permissions you no longer need — especially Screen Recording, Accessibility, and Network Access — to keep your system locked down.
