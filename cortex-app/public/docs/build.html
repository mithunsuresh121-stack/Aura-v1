Build Mode

Plan and execute code changes with AI-powered diffs, per-step review, and one-click undo.

Overview

Build Mode transforms CORTEX from a conversational assistant into an autonomous engineering agent. Instead of just suggesting code changes, CORTEX can plan, execute, and manage file modifications across your entire project. Every change is tracked, reviewable, and reversible.

Use Build Mode when you need to make substantial code changes — refactoring modules, adding features, fixing bugs across multiple files, or restructuring your project. It operates in distinct phases that give you full control over what gets changed and when.

Creating a Plan

Before touching any files, CORTEX enters Plan mode. It reads the relevant source files, analyzes your request, and produces a structured plan outlining every file that needs to change and what the change entails.

Describe your task in natural language — the more specific you are, the better the plan:

  • Be specific — "Add email validation to the login form" is better than "improve the login".
  • Mention files — If you know which files to target, include them in your description.
  • Set constraints — "Use Zustand for state management" or "Keep it compatible with IE11".

The plan includes a file list, change summary for each file, dependencies between changes, and a risk assessment. You can accept the plan as-is, ask CORTEX to revise it, or reject it entirely. No code is written until you explicitly approve.

Tip: Always review the generated plan before approving. CORTEX marks high-risk changes — deletions, dependency modifications, config file edits — with a warning badge so you can spot potentially dangerous operations at a glance.

Executing Changes

Once you approve a plan, CORTEX executes it one file at a time. Each step produces a diff showing exactly what changed. Files are backed up automatically before any modification.

At each step you can:

  • Apply — Accept the change and move to the next step.
  • Skip — Leave the file unchanged and continue.
  • Edit — Modify the proposed change before applying it.
  • Abort — Cancel the entire build and revert all applied steps.
Note: CORTEX automatically creates backup copies of every file before modifying it. Backups are stored under .aura-build-backups/ in your project root. If something goes wrong, you can restore any file to its pre-build state from the History panel.

Viewing History

Every build session is recorded in the Build History panel. You can browse past sessions, inspect the full diff for each step, and see the exact state of every file before and after the build. History is organised chronologically and is searchable by file name or change description.

Each history entry shows the plan that was approved, every diff that was applied or skipped, and the final status of the build. You can jump back to any session and inspect its details at any time.

Undoing Changes

Build Mode supports one-click undo for the most recent change. Click the Undo Last button in the Build Panel to revert the last file modification. The original file is restored from its automatic backup.

Backups are kept for every modified file. If you need to restore a file from a previous session, open the History panel, locate the change entry, and click Restore. The backup copy will replace the current file.

Example Workflow

Below is a full walkthrough of a typical Build Mode session — from plan to finish.

1. Describe the change

In the chat panel, switch to Build Mode and describe what you want:

$ /build Add input validation to the login form
# CORTEX reads src/components/LoginForm.jsx and related files
# Produces a plan with 3 file changes

2. Review the plan

CORTEX presents the plan in a structured card:

# ── Plan ──────────────────────────────────
# File 1: src/components/LoginForm.jsx
#   → Add email format validation (regex)
#   → Add required-field check on submit
# File 2: src/styles/forms.css
#   → Add .input-error and .error-message styles
# File 3: src/i18n/en.json
#   → Add 2 new error message keys
# Risk: Low ✅

3. Approve and execute

Click Apply Plan. CORTEX works through each file, showing diffs as it goes:

# ── Applying: LoginForm.jsx ───────────────
#  +18  -4  src/components/LoginForm.jsx
# ── Diff ──────────────────────────────────
#  const validateEmail = (email) => {
# +  const re = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
# +  return re.test(email);
#  };
# 
# [Accept]  [Edit]  [Skip]  [Abort All]

4. Iterate or finish

After all files are processed, CORTEX summarises the build. You can then start another build, review history, or undo individual steps: 

# ── Build Complete ─────────────────────────
# 3 files changed, 24 insertions, 6 deletions
# Status: All changes applied successfully
# 
# [View History]  [Undo All]  [New Build]
