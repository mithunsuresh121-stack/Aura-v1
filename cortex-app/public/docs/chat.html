Multi-Model Chat

Chat with local and cloud models side-by-side, switch providers mid-conversation, and activate agent mode for autonomous tool use. CORTEX preserves context across model switches and persists every conversation for later retrieval.

Switching Models

CORTEX integrates with a wide range of model providers including OpenAI (GPT-4o-mini, GPT-4o), Google Gemini (2.0 Flash), Ollama (any local model), and the built-in GPT-2 fallback. You can switch between these providers at any point during a conversation without losing any of the message history.

The model picker is located in the top-left corner of the chat panel. Click it to expand a dropdown listing every configured provider and all available models for that provider. Selecting a new model routes subsequent messages through that provider while the conversation scroll remains intact. Any model can be used for any message — CORTEX does not enforce provider-locking per conversation.

Provider configuration is managed through the Settings panel. You can add API keys for cloud providers, configure Ollama endpoints, and set default models per provider. Each provider can expose multiple models; the picker lists them all. Models are loaded lazily — switching to an Ollama model triggers a pull if the tag is not already cached locally.

Context preservation works by storing every message in a provider-agnostic format. When you switch providers, the full conversation history is replayed as context for the new model. Cloud models with larger context windows (128K tokens on GPT-4o, 200K on Claude 3 Opus) can ingest longer conversations. If the conversation exceeds the target model's context limit, CORTEX automatically truncates the oldest messages while preserving the system prompt and the most recent exchanges.

$ OpenAI       GPT-4o-mini  |  GPT-4o
$ Google       Gemini 2.0 Flash
$ Ollama       llama3  |  mistral  |  codellama  |  (custom)
$ Local        GPT-2 (default, no API key needed)

Agent Mode

Agent mode elevates CORTEX from a passive chat assistant to an autonomous agent capable of reading files, running shell commands, searching the web, modifying code, and executing multi-step plans. Toggle agent mode on or off with the button in the chat toolbar or by typing /agent as the first line of your message.

When agent mode is active, CORTEX follows a structured reasoning loop:

  1. Plan — The model analyses your request and decomposes it into discrete steps. Each step specifies a tool to invoke and the expected outcome.
  2. Execute — CORTEX invokes tools one at a time, streaming the results back into the conversation. Supported tools include read_file, write_file, edit_file, run_command, web_search, web_fetch, grep, and glob.
  3. Review — After each tool call, the model evaluates whether the result matches the plan. If a step fails, CORTEX retries with an adjusted strategy or reports the failure with an explanation.
  4. Synthesise — Once all steps are complete, the model produces a summary of what was done, including any diffs applied, commands run, or files created.

Agent mode works with any configured model, but cloud models generally produce more reliable multi-step plans. Local models via Ollama can handle simpler tool chains — single-file edits, grep searches, and short command execution — while GPT-4o and Claude 3 Opus excel at complex workflows involving multiple interdependent steps.

$ /agent Find all TODO comments in the src/ directory
$ # CORTEX runs: rg -n "TODO" src/ --type ts
$ # Returns: src/utils.ts:42, src/handler.ts:15, src/api.ts:88
$ # Then asks if you want it to resolve any of them

$ /agent Refactor the fetch call in api.ts to use error boundaries
$ # CORTEX reads api.ts, identifies the fetch block,
$ # writes a diff, and applies it with user confirmation
Tip: You can reference files directly in agent mode with @filename. CORTEX reads the file contents and injects them as context. Example: /agent @src/config.ts update the API endpoint to v2.

Conversation Management

Every message you send is automatically saved to local storage. CORTEX organises conversations in a sidebar list sorted by last activity. The management interface gives you full control over your conversation history:

  • Save — Conversations are saved automatically as you type. There is no manual save button. Each message is persisted immediately after the response is received.
  • Rename — Click the pencil icon next to a conversation name in the sidebar to edit its title. CORTEX also auto-generates a title from the first message using the active model.
  • Delete — Click the trash icon to permanently remove a conversation. A confirmation dialog prevents accidental deletion. Deleting is irreversible.
  • Clear context — The "New Chat" button starts a fresh conversation without clearing your history. Old conversations remain accessible in the sidebar.
  • Export — The overflow menu provides "Copy as Markdown" and "Download as Markdown" options. The exported file includes every message with the model label, timestamp, and full content.
  • Search — The conversation sidebar includes a search bar. Type to filter conversations by title or message content (useful when you have dozens of saved chats).
Note: Conversation data is stored in your browser's local storage under the cortex-conversations key. Clearing browser data will remove all saved chats. Export important conversations before clearing.

Keyboard Shortcuts

CORTEX provides the following keyboard shortcuts for power users:

  • Cmd+Enter — Send message
  • Escape — Cancel / stop streaming
