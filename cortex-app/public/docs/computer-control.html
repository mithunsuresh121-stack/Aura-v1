Computer Control

Let CORTEX see your screen, move the mouse, type text, and run scripts — turning natural language into direct computer actions.

Overview

Computer Control gives the model the ability to interact with your operating system. When you enable this capability, CORTEX can capture your screen, analyze what it sees, and perform actions like clicking buttons, filling out forms, or executing terminal commands.

This is powered by a lightweight accessibility and screen-capture layer running on the Python server. All processing stays on your machine — nothing is ever sent to the cloud unless you explicitly use a remote model.

Screen Recording

CORTEX can capture screenshots of your display or individual windows. The model uses these captures to understand the current UI state and decide what actions to take. Screenshots are held in memory only and are never transmitted over the network.

You can request a screen capture explicitly or let CORTEX decide when it needs to look at the screen:

$ "What's on my screen right now?"
$ "Take a screenshot of the active window."
$ "Read the text from the dialog that just popped up."
Privacy: Screen captures are ephemeral. They exist only in memory during the active session and are discarded the moment they are no longer needed. CORTEX never stores or logs raw screen data.

Mouse and Keyboard

Once CORTEX sees the screen, it can move the cursor, click, drag, and type. This allows it to interact with any application just like a human would — no API integrations needed.

$ "Click the 'Continue' button in the installer."
$ "Type my email into the login field and press Enter."
$ "Select all the text in the editor and delete it."
$ "Drag the file icon onto the upload area."
Safety: Every mouse and keyboard action is logged in the activity panel. You can pause or cancel any action mid-execution. Use the Undo button to reverse the last action sequence if something goes wrong.

Script Execution

CORTEX can run shell scripts, AppleScript, and keyboard shortcuts. This enables deeper automation such as installing packages, modifying system settings, or orchestrating multi-step workflows across applications.

$ "Run 'brew update' in the terminal."
$ "Open System Settings and change the wallpaper."
$ "Create a new folder on the Desktop called 'Screenshots'."
$ "Run this AppleScript to resize all open windows."

Script execution respects the permission boundaries you set. By default, destructive commands (like rm, sudo, or file deletion) require explicit confirmation before they run.

Prerequisites

Accessibility Permission

On macOS, Computer Control uses the Accessibility API to move the mouse, send keystrokes, and capture screen content. You must grant this permission when prompted or manually in System Settings → Privacy & Security → Accessibility.

CORTEX will show a permission banner if the required access has not been granted. The feature will remain disabled until the permission is approved.

Tip: After granting Accessibility permission, restart the CORTEX server for the changes to take effect. You only need to do this once.

Screen Recording Permission

If you want CORTEX to capture screenshots, you may also need to enable Screen Recording permission under the same Privacy & Security pane. This is separate from the Accessibility permission and must be granted independently.

Common Use Cases

# Install Homebrew automatically
$ "Open the terminal and run the Homebrew install script."
$ "Type my password when prompted and press Enter."
# Fill out a web form
$ "Go to the signup page at example.com."
$ "Fill in 'john@example.com' and 'password123' and click Sign Up."
# Bulk rename files
$ "Rename all .png files in the Downloads folder to screenshot_1, screenshot_2, etc."
Note: All automated actions are previewed in the chat panel before execution unless you enable auto-execute. Review each action carefully — especially script commands that modify the filesystem.
Tip: You can combine Computer Control with Orchestration to decompose complex multi-step tasks. CORTEX will plan, execute, and verify each phase automatically.
